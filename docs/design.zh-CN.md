# Library Management System 设计文档（简体中文）

## 1. 文档目的与范围
本文档基于当前仓库实现（C++11 控制台版本）整理系统设计，覆盖：
- 系统目标与边界
- 架构与模块职责
- 核心数据模型与持久化
- 关键业务流程
- 安全、配置、报表、备份机制
- 已知设计风险与改进方向

## 2. 系统概览
### 2.1 产品定位
本系统是本地运行的图书馆管理系统，面向管理员与普通会员，支持：
- 认证与角色区分
- 图书/会员/借阅/预约管理
- 推荐与报表
- 数据备份与恢复

### 2.2 技术栈
- 语言：C++11
- 构建：CMake（`CMakeLists.txt`）
- 密码学：OpenSSL（PBKDF2-HMAC-SHA256）
- 存储：CSV 文件
- 交互：CLI（控制台菜单）

### 2.3 运行形态
- 单进程、单机、无需网络服务
- 业务状态常驻内存，按操作回写 CSV
- 首次启动自动种子数据（默认管理员、默认会员、示例图书）

## 3. 目录与分层
```
src/
  authentication/   认证与密码哈希
  config/           全局配置与可持久化设置
  models/           领域实体与CSV序列化
  managers/         业务管理器（CRUD+规则+持久化）
  ui/               终端UI与菜单流程
  utils/            文件、日期、输入校验工具
```

分层关系：
- `UI/MenuHandler` 调用 `managers`
- `managers` 使用 `models + utils`
- `authentication` 由 `Member/MemberManager` 使用
- `Config` 被主流程及部分管理器引用

## 4. 核心模块设计
### 4.1 配置模块（Config）
职责：
- 提供数据路径、业务常量、UI参数
- 维护运行时可调配置（高级UI、借期、罚金、默认借阅上限）
- 从 `data/settings.csv` 读写配置

关键点：
- 单例模式 `Config::getInstance()`
- 默认值初始化后可覆盖
- 提供 ID/类别相关辅助函数

### 4.2 认证模块（auth）
职责：
- 密码哈希：随机 salt + PBKDF2（100000 次）
- 密码校验：按存储格式分离 salt 与 hash 后比对

存储格式：
- `salt(16B hex 32字节) + hash(32B hex 64字节)`，总长 96

### 4.3 领域模型（models）
- `Book`：ISBN、书名、作者、出版社、类别、馆藏/可借数量、预约标记
- `Member`：ID、姓名、手机、偏好类别列表、注册/到期、借阅上限、管理员标记、密码哈希
- `Transaction`：交易ID、会员ID、ISBN、借出/到期/归还日期、续借次数、罚金、归还标记
- `Reservation`：预约ID、会员ID、ISBN、预约日期、有效标记

统一能力：
- `toCSV()/fromCSV()` 做序列化/反序列化

### 4.4 文件与日期工具（utils）
- `FileHandler`：CSV 读写、存在性检查、目录创建、简单缓存
- `DateUtils`：日期与时间戳转换、当前日期时间、日期增减、天数差
- `Validator`：ISBN/手机号/ID等校验

### 4.5 业务管理器（managers）
#### BookManager
- 维护图书集合与持久化
- 支持增删改查、按字段检索、借还库存变更
- 支持批处理 `BatchOperation`（关闭自动保存，析构时统一落盘）

#### MemberManager
- 维护会员集合与持久化
- 支持增删改查、按字段检索、管理员筛选
- 提供 `authenticateUser` 登录校验
- 支持批处理保存

#### TransactionManager
- 维护借阅交易集合与持久化
- 支持借书/还书/续借、活跃与逾期检索
- 交易ID按 `T + 年 + 季度 + 5位流水` 生成
- 借阅前检查：会员存在与有效、未超借阅上限、图书可借
- 罚金规则：逾期按天计费并封顶（模型层固定 2.0/天，上限14.0）

#### ReservationManager
- 维护预约集合与持久化
- 每个 ISBN 维护 FIFO 队列（`map<string, deque<string>>`）
- 预约ID按 `R + 年 + 季度 + 5位流水` 生成
- 创建/取消预约时同步图书 `isReserved` 标记
- 提供排队位置、队列长度、下一位等查询

#### RecommendationManager
- 混合推荐：
1. 用户向量（偏好+历史）
2. 会员间余弦相似度（KNN）
3. 候选书打分（邻居协同 + 热度）
4. 冷启动回退（内容偏好 + 热度）
- 支持可借过滤与 TopN

#### ReportManager
- 聚合四大管理器数据生成文本报表：
1. Summary
2. Inventory
3. Member
4. Transaction
5. Reservation
6. TopBorrowedBooks
- 文件输出默认 `reports/`

#### BackupManager
- 备份数据文件至 `data/backup/<backupID>/`
- 维护 `backup_manifest.txt`
- 支持恢复、列出、获取最新、有效性检查、保留最近 N 份清理

### 4.6 UI 与菜单控制
- `UI`：封装简单/高级显示模式、输入、消息与渲染
- `MenuHandler`：
- 登录/登出流程
- 会员菜单：检索、借还续借、预约、推荐、个人信息
- 管理员菜单：图书/会员/交易/预约管理、报表、备份恢复、系统设置

## 5. 数据存储设计
### 5.1 CSV 文件
- `data/books.csv`
- `data/members.csv`
- `data/transactions.csv`
- `data/reservations.csv`
- `data/settings.csv`

### 5.2 读写策略
- 管理器初始化时加载内存
- 默认每次写操作立即落盘（autoSave）
- 批处理模式延迟保存，结束时一次性落盘

### 5.3 一致性特点
- 当前无事务机制，跨管理器写入失败时采用部分回滚逻辑（例如借书失败删除交易）

## 6. 关键业务规则
- 默认借期：14天
- 续借：每次 +7 天，总借期 <= 30 天
- 会员借阅上限：由会员字段控制
- 逾期罚金：2.0/天，封顶 14.0
- 预约队列：同书 FIFO
- 同会员同ISBN不允许重复有效预约
- 到期会员不可借阅/预约

## 7. 关键流程
### 7.1 启动流程
1. 初始化控制台编码与工作目录探测
2. 加载配置并保存默认配置文件
3. 初始化各管理器
4. 若会员/图书为空则注入种子数据
5. 进入登录与菜单循环

### 7.2 借书流程
1. 校验会员有效与额度
2. 校验图书可借
3. 创建交易记录
4. 图书可借数量 -1
5. 持久化

### 7.3 还书流程
1. 定位活跃交易
2. 图书可借数量 +1
3. 交易标记归还并计算罚金
4. 持久化

### 7.4 预约流程
1. 校验会员/图书存在与会员有效
2. 检查重复预约
3. 创建预约并入队
4. 更新图书预约标记
5. 持久化

### 7.5 备份恢复流程
- 备份：创建目录 -> 拷贝数据文件 -> 记录manifest
- 恢复：验证备份完整性 -> 覆盖数据文件

## 8. 安全与可靠性
### 8.1 已实现
- 密码不明文存储
- 基于角色的菜单权限分流

### 8.2 当前不足
- 无账户锁定/登录限速
- 无文件锁与并发保护

## 9. 已识别设计风险（基于代码）
1. `CMakeLists.txt` 设置 `cmake_minimum_required(VERSION 4.1)`，版本较高，可能导致环境兼容性问题。
2. 跨管理器复合写操作缺乏原子性保证。

## 10. 非功能特性评估
- 可移植性：Windows/Linux 基础支持
- 可维护性：模块划分清晰
- 可扩展性：管理器分层有利于后续数据库化/服务化
- 性能：数据量小场景足够，大规模数据受CSV和内存线性检索限制
