# Library Management System

## Overview
This project is a **C++11** console-based Library Management System that covers core workflows for books, members, borrowing transactions, reservation queues, recommendations, reporting, and data backup/restore.

The system stores data in CSV files, supports cross-platform builds (Windows / Linux), and uses OpenSSL for password hashing.

## Key Features
- Authentication with role-based access (Admin / Member)
- Book management: create, update, delete, search, and availability control
- Member management: create, update, delete, search, and admin identity support
- Transaction management: borrow, return, renew, history, overdue detection, and fine calculation
- Reservation management: create/cancel/query reservations with FIFO queue position
- Recommendation engine: KNN + cosine similarity using preferences and borrowing history
- Reporting: summary, inventory, member, transaction, reservation, and top-borrowed reports
- Backup system: data snapshots, manifest management, restore, and old-backup cleanup
- UI modes: simple vs. advanced display style

## Tech Stack
- Language: C++11
- Build system: CMake
- Crypto: OpenSSL (PKCS5_PBKDF2_HMAC + SHA-256)
- Storage: CSV files

## Module Notes
- `models/`: domain objects (Book, Member, Transaction, Reservation) and CSV serialization
- `managers/`: business managers (CRUD, business rules, persistence)
- `authentication/`: password hashing and verification
- `config/`: constants and runtime settings

- `utils/`: date, file, and validation helpers

## Build and Run
### Prerequisites
- C++11-compatible compiler (GCC/Clang/MSVC)
- CMake
- OpenSSL development package

### Build
```bash
cmake -S . -B build
cmake --build build
```

### Run
```bash
./build/Library_Management_System
```

On Windows, it may look like:
```powershell
.\build\Debug\Library_Management_System.exe
```

## First-Run Behavior
When `data/*.csv` files are empty, startup logic seeds default admin/member accounts and sample books automatically.

Default credentials:
- Admin: `A20261001` / `admin123`
- Member: `M20261001` / `user123`

## Data Files
- `books.csv`: `ISBN,Title,Author,Publisher,Genre,TotalCopies,AvailableCopies,IsReserved`
- `members.csv`: `MemberID,Name,PhoneNumber,Preference,RegistrationDate,ExpiryDate,MaxBooksAllowed,IsAdmin,PasswordHash`
- `transactions.csv`: `TransactionID,MemberID,ISBN,BorrowDate,DueDate,ReturnDate,RenewCount,Fine,IsReturned`
- `reservations.csv`: `ReservationID,MemberID,ISBN,ReservationDate,IsActive`
- `settings.csv`: runtime parameters (UI mode, borrow period, fines, etc.)

## Business Rules (as implemented)
- Default borrow period: 14 days
- Renewal: +7 days each time, max total borrow period 30 days
- Overdue fine: `2.0/day`, capped at `14.0`
- Reservation queue policy: FIFO by reservation date
- Transaction/Reservation IDs: generated by `year + quarter + sequence`

## Report Output
`ReportManager` writes reports to `reports/` by default:
- SummaryReport
- InventoryReport
- MemberReport
- TransactionReport
- ReservationReport
- TopBorrowedBooksReport

## Backup and Restore
`BackupManager` uses `data/backup/` for backup folders and manifest files.
Supported operations:
- Create backup
- Restore by backup ID
- List backups
- Auto-clean old backups (keep latest N)

## Known Notes
- `CMakeLists.txt` currently sets `cmake_minimum_required(VERSION 4.1)`, which is unusually high; adjust for your environment if needed.
- Recommended to use UTF-8 encoding.
- `data/*.csv` being empty is normal before first successful startup; base data is seeded automatically.
